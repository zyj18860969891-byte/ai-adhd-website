# 提醒功能增强指南

## 功能概述

为提醒系统添加了两个新的操作选项：
1. ✅ **已完成** - 标记任务完成，状态变更为"completed"
2. ⏰ **延期** - 设置新的截止时间，创建新的提醒

## 功能特性

### 1. 提醒界面操作按钮

当提醒弹出时，用户可以看到两个按钮：

- **已完成** (绿色按钮)
  - 将任务状态更新为"completed"
  - 自动确认当前通知
  - 显示成功消息

- **延期** (黄色按钮)
  - 打开延期对话框
  - 提供多个延期选项
  - 创建新的提醒

### 2. 延期选项

延期对话框提供以下选择：

- 🕐 **1小时后** - 延期1小时
- 🕐 **2小时后** - 延期2小时
- 🕐 **4小时后** - 延期4小时
- 🌅 **1天后** - 延期1天
- ✏️ **自定义小时数** - 输入任意小时数

### 3. 自动化流程

#### 完成任务流程
```
用户点击"已完成"
    ↓
更新任务状态为"completed"
    ↓
确认当前通知
    ↓
显示成功消息
    ↓
刷新通知列表
```

#### 延期任务流程
```
用户点击"延期"
    ↓
打开延期选项对话框
    ↓
用户选择延期时间
    ↓
更新任务截止日期
    ↓
创建新的提醒（截止前1分钟）
    ↓
确认当前通知
    ↓
显示新提醒时间
    ↓
刷新通知列表
```

## 使用示例

### 场景1：任务已完成

1. 收到提醒："任务 '测试提醒功能6' 即将到期！"
2. 点击"已完成"按钮
3. 任务状态自动变更为"completed"
4. 显示消息："任务已完成！"

### 场景2：任务需要延期

1. 收到提醒："任务 '测试提醒功能6' 即将到期！"
2. 点击"延期"按钮
3. 选择延期选项（如"2小时后"）
4. 任务截止时间更新为2小时后
5. 创建新的提醒（2小时后的截止前1分钟）
6. 显示消息："任务已延期至 2025-12-06 17:12，新提醒将在 2025-12-06 17:11 触发"

## API 接口

### 延期提醒接口

**POST** `/api/services/notification/reminders/defer`

请求体：
```json
{
  "userId": "user-1",
  "taskId": "task_123",
  "title": "任务提醒: 测试任务",
  "message": "任务 '测试任务' 即将到期！",
  "newSchedule": "11 17 6 12 6",  // cron格式
  "deferHours": 2
}
```

响应：
```json
{
  "id": "reminder_1234567890_deferred",
  "userId": "user-1",
  "taskId": "task_123",
  "title": "延期提醒: 任务提醒: 测试任务",
  "message": "任务 '测试任务' 即将到期！ (已延期 2 小时)",
  "schedule": "11 17 6 12 6",
  "repeat": false,
  "active": true,
  "createdAt": "2025-12-06T07:10:00.000Z"
}
```

## 注意事项

1. ✅ 延期功能会自动创建新的提醒
2. ✅ 任务状态更新是实时的
3. ✅ 通知会自动确认和刷新
4. ⚠️ 延期时间必须大于0
5. ⚠️ 需要确保服务端时区设置正确（北京时间）
