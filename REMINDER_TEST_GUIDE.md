# 实时任务提醒功能测试指南

## 问题分析

根据日志分析，发现以下问题：

1. **连接错误**: `ECONNREFUSED ::1:3003` - 通知服务端口配置错误
2. **数据库字段解析问题**: `tags` 和 `context` 字段存储为字符串而不是 JSON 对象
3. **通知服务端口**: 通知服务应该运行在 8380 端口而不是 3003

## 已修复的问题

### 1. 修复通知服务端口
- 将通知服务端口从 3003 改为 8380
- 更新数据库服务中的连接 URL

### 2. 修复数据库字段解析
- 在所有 API 响应中正确解析 JSON 字段
- `tags` 字段: 从字符串解析为数组
- `context` 字段: 从字符串解析为对象
- `details` 字段: 从字符串解析为对象

### 3. 修复通知服务的 ID 生成
- 确保通知有正确的 ID 字段
- 正确设置通知到 Map 中

## 如何测试提醒功能

### 方法 1: 手动测试

1. **创建任务**:
   - 在任务管理界面创建一个新任务
   - 设置截止日期为 1-2 分钟后
   - 例如: `2025-12-05 19:20` (当前时间 + 1 分钟)

2. **等待提醒**:
   - 系统会在截止日期前 1 分钟自动创建提醒
   - 提醒会在任务到期前 1 分钟显示

3. **检查提醒**:
   - 查看任务列表右上角的红色角标
   - 角标数字表示即将到期的任务数量
   - 点击任务可以查看详情

### 方法 2: 使用测试脚本

运行测试脚本验证功能:

```bash
node test-reminder.js
```

### 方法 3: 使用 curl 命令

```bash
# 创建任务
curl -X POST http://localhost:8280/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "id": "test_task_123",
    "title": "测试提醒",
    "description": "测试任务",
    "category": "工作",
    "priority": "高优先级",
    "dueDate": "'$(date -d '+2 minutes' '+%Y-%m-%dT%H:%M')'",
    "tags": ["测试"]
  }'

# 检查通知
curl http://localhost:8380/api/notifications/user-1
```

## 预期行为

1. **任务创建时**:
   - 如果设置了截止日期，系统自动创建提醒
   - 提醒时间 = 截止日期 - 1 分钟

2. **提醒触发时**:
   - 任务列表右上角显示红色角标
   - 角标数字 = 即将到期的任务数量
   - 任务卡片高亮显示

3. **查看提醒**:
   - 点击任务卡片查看详情
   - 可以标记任务为完成状态

## 常见问题排查

### 1. 提醒没有出现
- 检查截止日期是否设置正确
- 确认截止日期是未来时间
- 检查日志中的错误信息

### 2. 连接错误
- 确认所有服务都在运行
- 检查环境变量配置
- 验证端口配置正确

### 3. 数据库字段问题
- 确认 tags 和 context 字段正确解析
- 检查 API 响应格式
